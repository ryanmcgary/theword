The Problem

  • <video> doesn't work in old browsers
  • Browsers have inconsistent player UI
  • You can't use a single media file
    H.264 doesn't work in Firefox & Opera.
    WebM doesn't work in phones or Safari.

The Solution: MediaElement.js

If your browser doesn't support <video> or the media file provided, MediaElement.js replaces <video> with a Silverlight or Flash plugin wrapped in some JavaScript that matches the HTML5 Media Element API. This lets you code to MediaElement.js as if it were a native <video>.

Bonus! MediaElementPlayer.js

MediaElementPlayer.js uses MediaElement.js and jQuery to build the complete working player you see here. The same HTML/CSS controls show up for everyone on any browser with any media type (use a single H.264 or include WebM, Ogg, WMV, FLV). It even works on iPhones, iPads, and Androids.

Browser support (using H.264)

  • iPhone, iPad: native HTML5
  • Android: native HTML5
  • Safari: native HTML5
  • Chrome: native HTML5
  • Internet Explorer 9: native HTML5
  • Internet Explorer 6,7,8: Silverlight/Flash
  • Firefox: Silverlight/Flash
  • Opera: Silverlight/Flash
  • (Add OGG or WebM for native HTML5 in Firefox and Opera)

The Code

<script src="jquery.js"></script>
<script src="mediaelement.js"></script>
<script src="mediaelementplayer.js"></script>
<link href="mediaelementplayer.css" rel="Stylesheet" />

<!-- Preferred: Use one H.264 file for everyone (with silverlight fallback) -->
<video src="myfile.mp4" type="video/mp4" width="640" height="360"></video>

<!-- OR use multiple codecs, your choice -->
<video width="640" height="360">
        <source src="myfile.mp4" type="video/mp4" >
        <source src="myfile.ogg" type="video/ogg" >
        <source src="myfile.webm" type="video/webm" >
</video>

<script>
// do NOT put this in the [head] tag or iOS 3.x devices will fail
jQuery(document).ready(function($) {
        $('video').mediaelementplayer();
});
</script>

JavaScript Options

MediaElementPlayer Options
Name Function
videoWidth overrides the video tag's attribute. default: -1
videoHeight overrides the video tag's attribute. default: -1
audioHeight the height of the player for audio tags. default: 30
audioWidth the width of the player for audio tags. default: 300
messages playback messages which can be changed for different languages
... Also takes all of the options of MediaElement below
MediaElement Options
Name Function
enablePluginDebug writes debug information to flash or silverlight. default: false
plugins array which determines the available fallback plugins and their priority. default: ['silverlight','flash']
type when you are dynamically loading video and don't have a default type, this tells MediaElement what to expect later. default: (blank)
flashUrl URL and name of flash fallback. default: flashmediaelement.swf
silverlightUrl URL and name of silverlight fallback default: silverlightmediaelement.xap
success Callback method when the API is ready (after Flash/Silverlight have been embedded). Parameters: sends MediaElement object which is the native media object or the wrapper for Flash/Silverlight. The properties, methods, and events are listed to the left.
error Callback method when something goes wrong.
